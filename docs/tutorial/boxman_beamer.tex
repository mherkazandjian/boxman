\begin{frame}{What is Boxman?}

\begin{itemize}
    \item A CLI wrapper around \textbf{libvirt}
    \item Infrastructure-as-Code for virtual machine clusters
    \item YAML-driven project configuration
    \item Full lifecycle management:
    \begin{itemize}
        \item Provision
        \item Start / Stop
        \item Snapshot / Restore
        \item SSH / Cluster execution
    \end{itemize}
\end{itemize}

\medskip
\textbf{Goal:} Reproducible, scriptable VM infrastructure.

\end{frame}

% ------------------------------------------------
\begin{frame}{Basic Workflow}

\begin{enumerate}
    \item Prepare configuration (see \texttt{boxes/} directory in project root)
    \item Provision infrastructure
    \item Start cluster
    \item Work / SSH / Run tasks
    \item Snapshot for safe experimentation
    \item Shutdown or destroy
\end{enumerate}

\medskip
Boxman keeps your VM environments consistent and versionable.

\end{frame}

\begin{frame}[fragile]{Install libvirt (APT-based distros)}

For Ubuntu / Debian:

\begin{lstlisting}
sudo apt update
sudo apt install -y \
  qemu-kvm libvirt-daemon-system \
  libvirt-clients virtinst \
  bridge-utils
\end{lstlisting}

Enable and start libvirt:

\begin{lstlisting}
sudo systemctl enable --now libvirtd
\end{lstlisting}

Verify:

\begin{lstlisting}
virsh list --all
\end{lstlisting}

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Install libvirt (DNF-based distros)}

For Fedora / RHEL / Rocky / Alma:

\begin{lstlisting}
sudo dnf install -y \
  @virtualization \
  libvirt virt-install
\end{lstlisting}

Enable and start:

\begin{lstlisting}
sudo systemctl enable --now libvirtd
\end{lstlisting}

Verify:

\begin{lstlisting}
virsh list --all
\end{lstlisting}

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Install Python + Virtual Environment}

Install Python and venv support.

APT:

\begin{lstlisting}
sudo apt install -y python3 python3-pip python3-venv
\end{lstlisting}

DNF:

\begin{lstlisting}
sudo dnf install -y python3 python3-pip
\end{lstlisting}

We will use a virtual environment to isolate dependencies.

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Clone and Install Boxman}

Clone the repository:

\begin{lstlisting}
git clone https://github.com/mherkazandjian/boxman
cd boxman
\end{lstlisting}

Create virtual environment:

\begin{lstlisting}
python3 -m venv venv
source venv/bin/activate
\end{lstlisting}

Install dependencies and project:

\begin{lstlisting}
pip install -r requirements.txt
pip install .
\end{lstlisting}

Verify installation:

\begin{lstlisting}
boxman --version
\end{lstlisting}

\end{frame}

\begin{frame}{Using Tested Configurations}

Instead of writing your own config initially:

\medskip

\textbf{Use the tested configurations in:}

\begin{center}
\texttt{boxes/}
\end{center}

Each directory contains ready-to-use \texttt{conf.yml} files.

\medskip

Navigate into one and start working immediately.

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Provision Infrastructure}

\begin{lstlisting}
boxman provision
\end{lstlisting}

This will:
\begin{itemize}
    \item Create VMs via libvirt
    \item Apply configuration
    \item Register project state
\end{itemize}

Provisioning is idempotent — safe to re-run.

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Start and Inspect Cluster}

Start infrastructure:

\begin{lstlisting}
boxman up
\end{lstlisting}

Check status:

\begin{lstlisting}
boxman ps
\end{lstlisting}

Direct libvirt check:

\begin{lstlisting}
virsh list
\end{lstlisting}

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Access and Execute}

SSH into VM:

\begin{lstlisting}
boxman ssh --vm node1
\end{lstlisting}

Run a command on all nodes:

\begin{lstlisting}
boxman run --cluster "uptime"
\end{lstlisting}

Great for automation and quick testing.

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Snapshots for Safe Experimentation}

Create snapshot:

\begin{lstlisting}
boxman snapshot take --name clean-state
\end{lstlisting}

List:

\begin{lstlisting}
boxman snapshot list
\end{lstlisting}

Restore:

\begin{lstlisting}
boxman snapshot restore --name clean-state
\end{lstlisting}

Snapshots enable safe experimentation and rollback.

\end{frame}

% ------------------------------------------------

\begin{frame}[fragile]{Shutdown and Cleanup}

Suspend cluster:

\begin{lstlisting}
boxman down
\end{lstlisting}

Destroy completely:

\begin{lstlisting}
boxman deprovision
\end{lstlisting}

Remove Docker runtime (if used):

\begin{lstlisting}
boxman destroy-runtime -y
\end{lstlisting}

\end{frame}
